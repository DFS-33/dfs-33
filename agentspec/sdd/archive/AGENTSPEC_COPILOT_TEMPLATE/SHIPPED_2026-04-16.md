## Summary

O repo `btc-zero-prd-claude-code` foi transformado em um template AgentSpec genérico para GitHub Copilot. Todo conteúdo específico do projeto original (UberEats, invoice pipeline, GCP, Gemini, LangFuse) foi removido dos arquivos `.github/prompts/` e substituído por conteúdo universal. Qualquer equipe pode agora clonar o repositório, preencher os `{placeholders}` em `copilot-instructions.md`, e ter o framework AgentSpec + GitHub Copilot operacional imediatamente — com 5 agentes universais, 4 KBs de domínio, e 8 arquivos de exemplo para estender o template com especialistas do próprio stack.

## Lessons Learned

### Process

- O **SDD workflow completo (Brainstorm → Define → Design → Build → Ship)** em uma única sessão é viável quando o escopo é bem delimitado — as 5 perguntas do Brainstorm foram suficientes para capturar todas as decisões
- Executar as tarefas **em etapas sequenciais com git commit entre elas** (cleanup primeiro, depois conteúdo) preveniu conflitos e manteve o histórico legível

### Technical

- Ao **deletar arquivos de KB**, é obrigatório verificar todos os arquivos que os referenciam com `grep -r` — 4 arquivos em `examples/` tinham links quebrados que só apareceram na verificação final
- O padrão `{e.g., ...}` nos placeholders de `copilot-instructions.md` é seguro: o contexto deixa claro que são exemplos, não referências reais de projeto
- Arquivos em `examples/` devem **manter as referências ao stack original** — esse é o valor deles como referência; não há necessidade de genericizá-los

### Communication

- A pergunta **"Qual você me indica?"** do usuário na fase de Brainstorm é um sinal de que o usuário quer liderança técnica, não lista de opções — respondeu com recomendação clara e raciocínio
- O fluxo Brainstorm → Define → Design foi executado **sem retrabalho** porque cada fase registrou fielmente as decisões da anterior

### Tools

- **`grep -r` para verificação final** é indispensável — não confiar em memória sobre o que foi editado
- O padrão **`Write` para arquivos grandes + `Edit` para pequenas correções** foi eficiente; evitou re-leitura desnecessária de arquivos longos
